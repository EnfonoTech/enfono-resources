Brain MCP — Frappe Corpus Search

Brain is Enfono's knowledge retrieval system for Frappe/ERPNext development. It indexes 615+ Frappe repositories (official + community), 365k+ vector embedding points, and 56 anonymized client contexts — all searchable directly from Claude Code via MCP (Model Context Protocol).

What Brain provides

CapabilityWhat it means
Code search (ripgrep) Full-text search across every Frappe repo — find functions, DocTypes, hooks, patterns
DocType lookup Find where any DocType is defined, which apps reference it, field definitions
Hook discovery Find every hooks.py that registers a specific hook key
Semantic search Vector similarity search (Voyage embeddings) — find conceptually related code
Print format search Match against 481 Enfono print formats (bilingual, ZATCA, etc.)
Client context Per-client merged config for KSA/UAE/India regional defaults

Connecting Claude Code to Brain

Prerequisites

  • Claude Code installed (VS Code extension or CLI)
  • mTLS certificate from the admin: ~/.config/frappe-brain/sayanth@enfono.com.p12
  • Brain MCP endpoint accessible: https://brain.enfonoerp.com/mcp

Configuration

Add this to your Claude Code config (~/.claude/claude.json or VS Code settings):

{
  "mcpServers": {
    "frappe-brain": {
      "type": "url",
      "url": "https://brain.enfonoerp.com/mcp"
    }
  }
}

The mTLS certificate is configured separately. Contact the admin for your personal .p12 file.

Network required: Brain MCP runs on Enfono's VPS. You must have network access (VPN or office network) to reach brain.enfonoerp.com.

Available Tools

Once connected, Claude Code gains these 8 tools (all prefixed with mcp__frappe-brain__):

1. frappe_corpus_search

Ripgrep across all 615 Frappe repositories. Best for exact symbol/string searches.

# Example: find all implementations of on_submit in Sales Invoice
query: "def on_submit" in erpnext accounts sales_invoice

2. frappe_find_doctype

Locate a DocType definition and all its references across the corpus.

# Example: where is "Sales Invoice" defined, and who references it?
doctype: "Sales Invoice"

3. frappe_find_hook

Find every hooks.py that registers a given hook key.

# Example: which apps hook into Sales Invoice events?
hook: "doc_events[Sales Invoice]"

4. frappe_knowledge_search

Vector similarity search using Voyage embeddings. Best for conceptual queries where you don't know the exact function name.

# Example: find code related to ZATCA e-invoicing
query: "ZATCA Phase 2 XML generation with QR code"

5. frappe_get_print_format

Match a query against 481 Enfono print formats. Returns matching templates with metadata.

# Example: find bilingual Arabic/English print formats
query: "bilingual Arabic English tax invoice"

6. frappe_client_context

Get merged configuration for a specific client (anonymized). Useful for understanding regional defaults and custom requirements.

# Example: get context for a Saudi client
client: "ksa-retail"

7. frappe_capture_correction

Opt-in: when Claude makes a mistake in Frappe code, capture the correction to improve future responses.

8. frappe_capture_concept

Opt-in: capture a Frappe concept or pattern for future reference.

Integration with frappe-erpnext-expert skill

The frappe-erpnext-expert skill in Claude Code automatically uses Brain MCP tools. When you ask any Frappe-related question, the skill:

  1. Searches the corpus for prior art before writing custom code
  2. Shows existing solutions alongside custom options
  3. Validates patterns against semgrep-rules
  4. Applies client-specific regional defaults
Pro tip: When asking Claude Code a Frappe question, be specific: "How does Sales Invoice on_submit calculate outstanding amount?" works better than "explain sales invoice". The Brain MCP tools return targeted results that Claude uses to build precise answers.

Troubleshooting

Brain tools not showing up in Claude Code

  1. Verify the MCP server is in your Claude Code config
  2. Restart Claude Code after adding the config
  3. Check network: curl https://brain.enfonoerp.com/mcp should return a JSON error (not timeout)

Certificate errors

mTLS requires a valid .p12 certificate. If you see TLS errors:

  1. Check your cert exists: ls ~/.config/frappe-brain/
  2. Contact the admin for a new certificate if expired
  3. Certificates are per-user — don't share yours

Search returns no results

  • Try frappe_knowledge_search instead of frappe_corpus_search — conceptual search is broader
  • Check your spelling — ripgrep is exact match
  • For community apps, verify the repo is in the indexed list
