Support & AMC

SOP-ERP-002-C + SLA-ERP-001 · Internal Use Only. Life after go-live — hypercare, the Annual Maintenance Contract, the support process, and service levels. Targets below are the Enfono standard; the signed AMC contract may tune coverage, hours, and targets per client. Part C of the Project Delivery SOP.

Boundary: defects, configuration help, and break-fix on delivered scope are SUPPORT (covered by AMC). New features, modules, or scope beyond the signed BRD are CHANGE REQUESTS (separately quoted) — never absorbed silently as support.

1. Hypercare

First weeks live (Wk 10–12 or as contracted).

  • FCDaily 15-min client check-in during the first live week; handle queries + minor issues.
  • PM+FCRespond to Critical production issues within 2 business hours; all new requests → Change Request.
  • PMWeekly hypercare status summary to the client POC.

2. AMC Scope

Included (AMC support)Excluded (Change Request / separate)
Break-fix on delivered featuresNew features, modules, or DocTypes
Configuration help + minor tweaksScope beyond the signed BRD
Bug fixes + error investigationData migration of new datasets
Version / security patching (planned)Third-party integrations not in scope
User how-to + guidanceNet-new reports / print formats
Monthly health check + reportBulk training for net-new staff

AMC scope, duration, SLAs, and commercials are defined in the maintenance agreement signed at completion (tracked on the Closure Checklist).

3. Support Process

Intake▶ Triage + priority▶ Resolve (L1→L2)▶ Verify with client▶ Report
  • FCIntake + triage: log every request via the agreed channel, set priority, apply known fixes (L1).
  • TLResolve code/data issues (L2); deploy fixes via staging → backup → production.
  • PMOwn SLA adherence, escalations, and the monthly support report.
