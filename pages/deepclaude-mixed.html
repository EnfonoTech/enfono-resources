DeepClaude Mixed

DeepClaude Mixed is a local proxy that routes Claude Code through DeepSeek's models for reasoning-heavy tasks at a fraction of Anthropic's cost. One proxy, all tools — Claude Desktop, Claude Code CLI, VS Code, Cursor.

Why use it

  • Cost savings: DeepSeek is significantly cheaper than Anthropic for reasoning tokens
  • Flexible routing: Choose which model handles thinking vs. final output
  • Transparent: Claude Code thinks it's talking to Anthropic's API — the proxy handles the split
  • Local: Runs on your machine, no data leaves your control except to the respective APIs
  • VS Code extension: Zero env var setup — install extension, paste API key, done

Quick Start

Recommended: Use the VS Code extension for zero-config setup. Works identically on macOS, Windows, Linux, and WSL.

Option A — VS Code Extension (Recommended)

  1. Install extensions

    Ctrl+Shift+X → install both:

    • Claude Code (anthropic.claude-code)
    • DeepClaude Mixed (ENfono.deepclaude-mixed)

    Or install the .vsix manually:

    code --install-extension deepclaude-mixed-0.1.3.vsix

    Download from GitHub Releases.

  2. Set API key

    Ctrl+Shift+P → DeepClaude Mixed: Set DeepSeek API Key → paste your key → Enter.

    Key stored in OS keychain (macOS Keychain / Windows Credential Store / Linux libsecret). Never in plaintext.

  3. Done

    Status bar shows ✓ DeepClaude :3200. Open Claude Code (Ctrl+Shift+P → Claude Code: Open) and start using — all messages route through DeepSeek.

How it works

The extension starts the proxy as a VS Code subprocess, stores your API key in the OS keychain, and sets environment variables that the Claude Code extension inherits when it spawns the CLI.

Option B — CLI Install (for Claude Desktop / terminal users)

  1. Prerequisites

    Node.js 18+ — nodejs.org

    DeepSeek API key — platform.deepseek.com/api_keys

  2. Run installer

    npx deepclaude-mixed-setup

    Paste your DeepSeek key. Claude Desktop restarts. Proxy runs on 127.0.0.1:3200.

  3. Verify

    curl -s http://127.0.0.1:3200/_proxy/status

    Expected: "deepseekKey": "set"

Frappe Brain MCP Integration

The VS Code extension can configure Frappe Brain MCP into Claude Code. This gives Claude access to 615+ Frappe repos for code search, DocType lookup, and hook discovery.

  1. Get an API key from your admin
  2. Ctrl+Shift+P → Frappe Brain: Configure
  3. Paste your Brain API key → extension writes config to ~/.claude.json
  4. Restart Claude Code (or reload VS Code)
  5. Frappe Brain tools appear in Claude Code

For full Brain tool reference (all 8 tools, search syntax, troubleshooting), see the Brain MCP guide.

Config written:

{
  "mcpServers": {
    "frappe-brain": {
      "url": "https://brain.enfonoerp.com/mcp",
      "transport": "streamableHttp",
      "headers": { "x-api-key": "sk-brain-..." }
    }
  }
}
Independent systems: Brain and proxy work independently. No brain key? Proxy routes through DeepSeek as normal. Brain commands only fire when you explicitly run them.

Architecture

Claude Code / VS Code Extension
  │
  │  POST http://127.0.0.1:3200/v1/messages
  │  (thinks it's talking to Anthropic API)
  ▼
DeepClaude Mixed Proxy (localhost:3200)
  │
  ├─ Thinking phase → DeepSeek API (api.deepseek.com)
  │   Uses DEEPSEEK_API_KEY
  │
  ├─ Strips thinking from DeepSeek response
  │
  └─ Final response phase → Anthropic API (api.anthropic.com)
      Uses ANTHROPIC_API_KEY (or ANTHROPIC_BASE_URL)
  │
  ▼
Response returned to Claude Code (combined)

Claude Code CLI

If not using the VS Code extension, set env vars manually:

macOS / Linux

# Add to ~/.zshrc or ~/.bashrc
export ANTHROPIC_BASE_URL=http://127.0.0.1:3200
export ANTHROPIC_API_KEY=unused

# Reload
source ~/.zshrc

Windows

Set via System Environment Variables dialog (not PowerShell $env:):

  1. Press Win → type "environment variables"
  2. Click Environment Variables...
  3. Add: ANTHROPIC_BASE_URL = http://127.0.0.1:3200
  4. Add: ANTHROPIC_API_KEY = unused
  5. Restart Terminal/VS Code

Cursor

DeepClaude Mixed works with Cursor — same proxy, same port.

  1. Open Cursor → Settings → Models
  2. Set Anthropic API Key: unused
  3. Set Anthropic Base URL: http://127.0.0.1:3200
  4. Model names: claude-opus-4-6, claude-sonnet-4-6, claude-haiku-4-5-20251001
VS Code extension works in Cursor too: cursor --install-extension deepclaude-mixed-0.1.3.vsix. Same extension host, same SecretStorage.

Upgrading

npx deepclaude-mixed-setup@latest

Re-deploys latest proxy script, restarts service. Your API key preserved.

VS Code extension: download latest .vsix from GitHub Releases.

Uninstalling

npx deepclaude-mixed-uninstall

Removes: proxy script, auto-start service. Remove env vars from shell profile / system variables manually. Remove VS Code extension via Ctrl+Shift+X → uninstall.

Troubleshooting

Port already in use

VS Code extension auto-handles this — it detects existing proxies and reuses them, or auto-increments to the next port (3201, 3202, ...).

For CLI users:

# Find what's using port 3200
lsof -i :3200           # macOS/Linux
netstat -ano | findstr :3200  # Windows

# Kill the process
kill -9 <PID>            # macOS/Linux
taskkill /PID <PID> /F   # Windows

Claude Code says "Connection refused"

The proxy isn't running. Start it: npx deepclaude-mixed-setup@latest

DeepSeek API errors

Check your DEEPSEEK_API_KEY is valid and has credits. Test directly:

curl https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY"

VS Code: Claude Code extension ignores proxy

Cursor shows "invalid API key"

Set API key to literal string unused. Proxy doesn't validate it — auth handled server-side via proxy's stored key.

400 error on second message (multi-turn)

Old proxy version. Run npx deepclaude-mixed-setup@latest. Fixed in v0.1.2.

WSL users

Recommended: Use the DeepClaude Mixed VS Code extension. Works automatically on WSL — no proxy install needed inside the WSL VM. Install extension in your VS Code WSL session, set API key, done.

Manual (if not using extension): Install proxy inside WSL (the Windows proxy at 127.0.0.1:3200 is not reachable from WSL's loopback):

npx deepclaude-mixed-setup   # inside WSL terminal
