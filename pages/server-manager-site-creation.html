Server Manager — Site Creation Do & Don't

When the manager shows an error during site creation, developers sometimes open a terminal and create the site manually via bench. This guide explains what is safe and what will break production.

One Rule: bench setup nginx is a nuclear option. Never run it on a managed server. Manager's nginx config has hand-crafted SSL blocks that bench doesn't know about. One command undoes hours of work.

✅ Safe to do in terminal

1. Create the site

su - v15 -c "cd frappe-bench && bench new-site sitename.enfonoerp.com \
  --admin-password admin \
  --mariadb-root-password <rootpass> \
  --install-app erpnext"

Replace v15 with the bench user for that server (e.g. frappe, hatco). Never run bench commands as root.

2. Install additional apps

su - v15 -c "cd frappe-bench && bench --site sitename.enfonoerp.com install-app hrms"

3. Migrate

su - v15 -c "cd frappe-bench && bench --site sitename.enfonoerp.com migrate"

4. Register site in manager (required)

After creating the site in bench, register it so the manager can track it and handle nginx/SSL:

SECRET=$(grep AGENT_SECRET /opt/server-manager-agent/.env | cut -d= -f2)

curl -s -X POST http://localhost:3847/api/sites \
  -H "Authorization: Bearer $SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "serverId": "<SERVER_ID>",
    "siteName": "sitename.enfonoerp.com",
    "benchPath": "/home/v15/frappe-bench"
  }'

Then use the Manager UI → server → site → Set Domain + SSL to configure nginx and issue the cert.

❌ Never do this

CommandWhy it breaks things
bench setup nginx Rewrites entire /etc/nginx/conf.d/frappe-bench.conf from bench's template — wipes all SSL blocks, shared certs, custom server blocks set up by the manager
bench setup production Calls bench setup nginx internally — same destruction, harder to notice
certbot --nginx -d newsite.com May merge new domain into an existing shared cert (changing its path) or issue a replacement cert — breaks SSL for every other site sharing that cert
certbot run or certbot renew without --cert-name Renews all certs, can restructure shared cert domains unexpectedly
Running any bench/certbot command as root Creates root-owned log files under sites/ — Frappe worker can't write logs → login fails with PermissionError

When manager shows an error

  1. Check job logs first

    Manager UI → Jobs → click the failed job → see the exact error step. Most errors are fixable without touching the terminal.

  2. Common fixable errors

    ErrorFix
    MariaDB root password wrongUpdate password in server settings, retry job
    App not in benchsu - v15 -c "bench get-app <app>" then retry via manager
    Disk fullClear space, retry
    DNS not propagatedWait for DNS, then retry SSL step only

  3. If manager is truly broken and site must go live NOW

    Use the safe terminal steps above (bench new-site + install apps). Stop there. Do not touch nginx or certbot. The site will work internally.

    Register the site in manager API, then use manager UI to add domain + SSL once the urgent issue is resolved.
