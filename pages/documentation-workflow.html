Documentation Workflow

Complete guide to creating, generating, and deploying documentation at Enfono. Covers two approaches: manual authoring with the docs-starter template, and auto-generation from Frappe custom app structure.

Choosing your approach

ApproachWhen to useTime to live
Manual authoring Internal team docs, guides, resources (like this site) ~5 minutes
docs-generate.sh Auto-generating docs from a Frappe custom app ~2 minutes

Approach 1: Manual Authoring

Step-by-step

  1. Create a new repo from the docs-starter template

    # Clone the starter
git clone https://github.com/EnfonoTech/docs-starter.git my-docs
cd my-docs

# Remove old git history and start fresh
rm -rf .git
git init
git add -A
git commit -m "Initial commit from docs-starter"

  2. Customize the pages

    Edit or replace pages in the pages/ directory. Keep the sidebar nav in sync:

    pages/
  index.html           ← Home page (main content at /index.html)
  my-guide.html        ← Your custom page
  another-guide.html   ← Another custom page

    Update the sidebar in each page to include links to your new pages.

  3. Style customization

    Edit css/style.css to change the theme. CSS variables at the top control colors:

    :root {
  --accent: #e94560;       /* Primary accent color */
  --sidebar-bg: #1a1a2e;  /* Sidebar background */
  --bg: #f5f6fa;          /* Page background */
}

  4. Push to GitHub

    gh repo create EnfonoTech/my-docs --public --source=. --remote=origin --push

  5. Deploy via Server Manager

    Go to Docs Hub → Create New Site:

    • Source: Existing repo → EnfonoTech/my-docs
    • Site name: my-docs
    • Branch: main

    Your site will be live at https://my-docs.docs.enfonoerp.com in ~30 seconds.

Approach 2: Auto-Generation from Frappe App

Use the docs-generate.sh script (part of the frappe-erpnext-expert skill) to automatically generate comprehensive documentation from any Frappe custom app's source code.

What it generates

PageSourceContent
overview.htmlpyproject.tomlApp name, version, description, module list, stats
doctypes.htmlDocType JSONsAll DocTypes with field tables (name, type, label, required)
modules.htmlmodules.txtModule structure with linked DocTypes
hooks.htmlhooks.pyRegistered hooks, whitelisted methods list
permissions.htmlDocType JSONsRole-permission matrix
api.htmlhooks.pyWhitelisted method endpoints

Usage

Dry run (inspect before publishing)

bash ~/.claude/skills/frappe-erpnext-expert/scripts/docs-generate.sh \
  /home/frappe/frappe-bench/apps/rmax

Generates docs in /tmp/docs-output/site/. Review the pages, then publish.

Publish (push to GitHub + deploy)

AGENT_SECRET=<secret> bash ~/.claude/skills/frappe-erpnext-expert/scripts/docs-generate.sh \
  /home/frappe/frappe-bench/apps/rmax \
  EnfonoTech/rmax-docs \
  --publish

What happens during --publish:

  1. Generates all 6 documentation pages from app source
  2. Clones EnfonoTech/docs-starter template
  3. Replaces template pages with generated content
  4. Creates (or updates) the specified GitHub repo
  5. Pushes generated docs to the repo
  6. Registers the site with Server Manager Docs Hub API
  7. Docs Hub auto-deploys from the GitHub repo

Requirements for publishing

RequirementHow to get it
AGENT_SECRETFrom Server Manager's .env file on the control server
gh CLI authenticatedgh auth login — needs repo creation permissions
App path existsFull path to the Frappe app on the bench server
GitHub repo nameowner/repo format, e.g. EnfonoTech/rmax-docs
Where to get AGENT_SECRET: 
ssh root@<control-server> 'grep AGENT_SECRET /opt/server-manager-agent/.env'

Linking Docs to Frappe Sites

You can link a documentation site to the Frappe production/dev sites it documents:

  1. Go to Docs Hub
  2. Click the edit icon on any site card
  3. In the "Linked Frappe Sites" section, check the boxes next to relevant sites
  4. Save — linked sites appear as chips on the site card

This connection is bidirectional: the linked Frappe site's detail page also shows the documentation link.

Post-Deploy: Setting Up Auto-Deploy

After your site is live, set up the GitHub webhook so it auto-deploys on every push:

  1. In Server Manager, click "Webhook Guide" on your site's card
  2. Follow the instructions to add a webhook in GitHub repo settings
  3. Webhook URL: https://docs.enfonoerp.com/webhook
  4. Use the webhook secret from the guide
  5. After setup, push a test commit — your changes go live in ~30 seconds

Detailed webhook setup: see Docs Hub & Webhooks.

Maintaining Docs with Claude Code

Every docs repo has a CLAUDE.md file that tells Claude how to edit pages. Tell Claude:

Update the getting-started page to include the new API endpoint

Claude reads CLAUDE.md, understands the page structure, edits the HTML, commits, and pushes. The webhook triggers deploy — your docs update automatically.

Pro tip: The CLAUDE.md in docs-starter contains instructions for Claude on how to maintain the site. Customize it for your project's specific conventions.
